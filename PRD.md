> **版本**: v1.2.0  
> **更新日期**: 2026-06-14  
> **产品类型**: AI SaaS Web应用  
> **目标市场**: 求职者（国内+海外）

---

## 1. 产品概述

### 1.1 一句话描述

Resume Matcher 是一款 AI 驱动的简历优化工具，帮助求职者根据职位描述(JD)智能定制简历、生成求职信、分析匹配度，提升简历通过率。

### 1.2 解决什么问题

求职者面临的核心痛点：
- **简历千篇一律**：同一份简历投所有岗位，缺乏针对性
- **不懂JD关键词**：看不懂招聘方真正看重什么技能
- **写求职信耗时**：每份定制化求职信需要30分钟以上
- **匹配度不可量化**：不知道自己与岗位的差距在哪里

### 1.3 产品价值主张

上传简历 + 粘贴JD → AI自动分析 → 生成定制化简历和求职信 → 导出PDF

**核心差异点**：
- 不只生成，更能量化匹配度（0-100分 6维度评分）
- 支持中/英/日/西 4语言
- AI简历润色——开放式问答引导而非一键填充

## 2. 目标用户

### 2.1 用户画像

| 画像 | 特征 | 核心需求 |
|------|------|---------|
| **海投型求职者** | 每天投递10+岗位，覆盖多行业 | 快速定制、批量管理多版本简历 |
| **转行求职者** | 跨行业投递，不了解目标行业关键词 | JD分析、技能差距识别 |
| **应届生** | 简历单薄、缺少工作经验描述技巧 | AI润色简历条目，增强表达 |
| **海外求职者** | 需要英文/日文简历 | 多语言简历模板、本地化表达 |

### 2.2 使用场景

1. **看到心仪岗位** → 粘贴JD → 分析匹配度 → 针对性修改简历 → 导出PDF投递
2. **跨行业海投** → 上传多份JD → 分析各行业技能要求 → 制定学习路径
3. **简历从零开始** → 上传原始简历 → AI润色每条经历 → 选择模板 → 导出精美PDF

## 4. 核心功能详述

### 4.1 AI简历定制 (Tailor)

**用户故事**：作为求职者，我想粘贴职位描述后让AI自动修改我的简历，使其更匹配该岗位。

**流程**：
1. 用户选择一份简历
2. 粘贴目标职位描述
3. 选择提示策略（标准/激进/保守）
4. AI生成改进预览，逐条展示原文→修改后对比
5. 用户逐条确认/拒绝/手动编辑
6. 确认后保存为新版本简历

**关键规则**：
- 不能无中生有（不编造经历）
- 保留用户原始语言的语种（中/英文简历分别用对应语言输出）
- 每条修改需标注原因

### 4.2 AI匹配分析 (Resume Match)

**用户故事**：作为求职者，我想知道我的简历与目标岗位的匹配程度，以及差距在哪里。

**评分维度**（0-100分）：
| 维度 | 权重 | 说明 |
|------|------|------|
| 技能匹配 | 30% | 技术栈/工具/证书匹配度 |
| 经验匹配 | 25% | 工作年限/项目经验相关度 |
| 教育背景 | 15% | 学历/专业匹配度 |
| 语言能力 | 10% | 语言要求达标度 |
| 项目经验 | 15% | 项目规模和角色匹配 |
| 综合匹配 | 5% | 整体印象分 |

**输出**: 总分 + 等级（优秀/良好/一般/较差）+ 差距列表 + 改进建议

### 4.3 JD分析 (JD Analysis)

**用户故事**：作为转行求职者，我想了解目标行业到底需要什么技能，以及我该如何学习。

**流程**：
1. 上传1-N份相关岗位JD
2. AI提取并分类技能（必备技能/加分技能/软技能）
3. 识别高频技能（出现次数排序）
4. 生成分阶段学习路径（入门→进阶→精通）
5. 推荐GitHub开源项目（匹配技能关键词）

**输出**: 技能雷达图 + 学习路径时间线 + 项目推荐列表

### 4.4 AI简历润色 (Enrichment)

**用户故事**：作为应届生，我的实习经历描述太简单，AI能帮我写出更专业的表达。

**流程**：
1. AI扫描简历，识别"薄弱条目"（描述过于简短、缺乏量化数据）
2. 逐条提问引导用户补充信息（STAR法则：情境/任务/行动/结果）
3. 根据用户回答，AI重写为专业表达
4. 也支持自由文本指令（如"把这个项目写得更偏管理方向"）

### 4.5 多语言支持

- UI界面：中文/英文/日文/西班牙文
- AI输出语言：自动检测用户简历语言，用对应语言输出
- 提示词：各语言独立配置

## 5. 技术方案概述

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **AI引擎** | LiteLLM（多提供商统一接口） | 支持DeepSeek/OpenAI/Anthropic等，可灵活切换 |
| **后端** | FastAPI + Python 3.13+ | 高性能异步、类型安全 |
| **前端** | Next.js 16 + React 19 | SSR/SSG混合渲染、App Router |
| **样式** | Tailwind CSS v4 | 原子化CSS、响应式友好 |
| **数据库** | TinyDB (JSON) + Neon PostgreSQL | TinyDB存储简历数据，PostgreSQL存用户认证 |
| **PDF渲染** | Playwright (Headless Chromium) | 完美还原HTML模板样式 |
| **认证** | Better Auth | 邮箱密码登录、Session管理 |
| **测试** | Vitest (112单元) + Playwright (15 E2E) | 覆盖所有API客户端和核心用户流程 |

### 5.1 AI提供商

默认使用DeepSeek (`deepseek-chat`) 以降低成本，支持通过设置页面切换为OpenAI/Anthropic等任何LiteLLM支持的提供商。

### 5.2 架构原则

- **前后端分离**：前端通过 `/api/*` 反向代理到后端(8001端口)
- **无状态API**：后端不存储会话状态，便于横向扩展
- **提示词与代码分离**：所有AI提示词存储在配置文件，可通过UI修改

## 6. 非功能需求

### 6.1 性能
- 简历上传响应 < 3秒
- AI分析结果返回 < 60秒
- PDF生成 < 10秒
- 首屏加载(LCP) < 2.5秒

### 6.2 安全
- 用户密码哈希存储（Better Auth自动处理）
- API密钥AES加密存储
- 简历数据按用户隔离
- Session 7天过期

### 6.3 兼容性
- 桌面：Chrome/Firefox/Edge/Safari 最新两个版本
- 移动端：响应式布局，支持iOS Safari/Android Chrome

### 6.4 可维护性
- 单元测试覆盖率 > 80%（API客户端）
- E2E测试覆盖核心流程（注册/登录/上传/定制）
- Git hooks（Husky）确保提交质量
